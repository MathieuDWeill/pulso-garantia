# Judging strategy

## The judging trap to avoid

Many hackathon projects use blockchain as a payment button. Pulso Garantia must show that Stellar changes the product logic.

## What to emphasize

### Integration depth and technical complexity

Say:

“Without the Soroban contract, this product cannot guarantee that money is reserved or released according to shared rules.”

Show:

- contract state machine;
- auth checks;
- testnet transaction proof;
- events;
- token transfer TODO or implementation.

### Impact on Stellar ecosystem

Say:

“Stellar becomes useful for everyday LATAM service commerce, not just crypto-native users.”

Show:

- Brazil/Argentina/Colombia scenario;
- USDC/XLM amount;
- small business use case.

### Customer discovery

Say:

“We did not start with escrow because it sounds crypto. We found a repeated trust gap in how people already work.”

Show:

- interview evidence links;
- quotes;
- current workflows.

### Testnet/mainnet deployment quality

Say:

“This demo is live on testnet. The repo includes instructions and proof links.”

Show:

- contract ID;
- transaction hash;
- README deploy section.

## Final pitch sentence

“Pulso Garantia makes Stellar the trust layer that lets LATAM buyers and sellers start working together sooner.”
